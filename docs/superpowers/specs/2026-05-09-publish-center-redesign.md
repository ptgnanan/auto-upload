# 发布中心页面重设计

> **日期**: 2026-05-09
> **范围**: `frontend/src/views/PublishCenter.vue` 及相关组件
> **目标**: 参考 docs/demo/ 截图，将现有双栏布局重设计为「左侧账号分组 + 右侧发布表单」的单列流式布局

---

## 1. 整体布局

采用 **左右分栏** 结构：

```
┌──────────────────────────────────────────────────────────┐
│  顶部操作栏：发布视频 | 保存草稿  浏览器发布  一键发布        │
├───────────────┬──────────────────────────────────────────┤
│               │                                          │
│  账号分组列表   │  发布内容区（可滚动）                       │
│  (固定 220px)  │                                          │
│               │  ┌ 公共配置 ──────────────────────────┐  │
│  ┌ 抖音 ─────┐│  │ 视频 | 封面                         │  │
│  │  用户A     ││  │ 标题  描述  标签                     │  │
│  │  用户B     ││  └───────────────────────────────────┘  │
│  └────────────┘│                                          │
│  ┌ 小红书 ────┐│  ┌ 个性化设置（渠道品牌色）────────────┐  │
│  └────────────┘│  │ 根据选中渠道动态显示字段              │  │
│  ┌ 快手 ──────┐│  │                                      │  │
│  └────────────┘│  └───────────────────────────────────┘  │
│  ...          │                                          │
│               │                                          │
│  [+ 添加账号]  │                                          │
└───────────────┴──────────────────────────────────────────┘
```

### 关键尺寸

| 区域 | 宽度 | 说明 |
|------|------|------|
| 左侧账号栏 | 220px 固定 | `flex-shrink: 0` |
| 右侧内容区 | 自适应剩余空间 | `flex: 1; min-width: 0` |
| 顶部操作栏 | 100% | 固定在内容区顶部，不随滚动 |

## 2. 左侧账号分组列表

### 2.1 分组结构

账号按渠道（平台）自动分组显示，无需切换。每个分组包含：

```
▼ [抖] 抖音                          2
    ● 小言爱听歌              (绿色状态点)
    ● 另一个抖音号             (绿色状态点)
▶ [红] 小红书                         2
▶ [快] 快手                           1
▶ [B]  哔哩哔哩                       1
▶ [视] 视频号                         0
```

- 分组头部：平台品牌色图标 + 平台名 + 账号数量 badge
- 展开/折叠：点击分组头部切换 ▼/▶
- 子账号：头像(22px) + 名称 + 状态点(绿色=正常, 红色=失效)
- 选中态：当前选中的分组有品牌色背景 + 边框；当前选中的账号有紫色边框头像

### 2.2 底部添加按钮

固定在列表底部，虚线边框样式：

```
┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐
       + 添加账号
└ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘
```

点击后弹出账号选择弹窗。

### 2.3 搜索（可选）

标题栏下方可加搜索框，快速过滤账号。

## 3. 右侧发布内容区

右侧内容根据左侧选中项动态变化，分为两层：

| 选中项 | 右侧显示 |
|--------|----------|
| 点击分组头部（如「抖音」） | 公共配置 + 该渠道的个性化设置 |
| 点击具体账号 | 公共配置 + 该账号所属渠道的个性化设置（预填该账号的值） |

### 3.1 顶部操作栏

```
发布视频  [抖音 · 个性化设置]          保存草稿  一键发布
```

- 标题「发布视频」
- 当前编辑的分组/账号标签（用品牌色标识当前渠道）
- 操作按钮：「保存草稿」（文字样式）、「一键发布」（紫色填充按钮）

### 3.2 公共配置区

所有渠道共享的内容，紫色竖线标识：

**模块 1：视频 + 封面（并排）**

| 区域 | 内容 |
|------|------|
| 视频 | 空态：「无视频 - 请选择视频」+ 「本地选择」「素材库」按钮 |
| 封面 | 空态：「无封面 - 请选择封面」+ 「剪裁」「本地上传」「从视频截取」按钮 |

已上传状态：显示缩略图 + 文件名 + 大小 + 删除按钮。

**模块 2：标题**

- `el-input` 输入框，带字数统计 `0/20`
- 占位符：「请输入标题...」

**模块 3：描述**

- `el-input type="textarea"` 多行输入，带字数统计 `0/1000`
- 占位符：「请输入描述...」

**模块 4：快捷标签按钮**

横向排列的标签按钮：
- `# 添加话题`
- `$ 参加活动`
- `@ 添加好友`
- `常用话题/标签`

点击弹出对应的标签选择弹窗。

### 3.3 个性化设置区

## 5. 数据模型

### 5.1 账号分组数据结构

```js
// 左侧分组数据（从 accountStore 派生）
const accountGroups = computed(() => {
  const groups = {}
  for (const account of accountStore.accounts) {
    const platform = account.platform
    if (!groups[platform]) {
      groups[platform] = {
        platform,
        name: platformIdToName[platform],
        accounts: [],
        expanded: false
      }
    }
    groups[platform].accounts.push(account)
  }
  return Object.values(groups)
})
```

## 7. 交互流程

1. 用户进入发布中心 → 左侧自动展开第一个有账号的分组
2. 点击分组头部 → 右侧显示公共配置 + 该渠道个性化设置（空表单）
3. 点击具体账号 → 右侧显示该账号的配置（如有缓存则恢复）
4. 编辑公共配置 → 对所有已选账号生效
5. 编辑个性化配置 → 仅对该渠道账号生效
6. 点击「一键发布」→ 验证必填项 → 构造请求数据 → 调用后端 API 发布
